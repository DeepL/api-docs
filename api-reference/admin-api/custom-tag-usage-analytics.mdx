The Admin API allows admins to retrieve usage statistics broken down by custom tags through the [Get custom tag usage analytics](/api-reference/admin-api/get-custom-tag-usage-analytics) endpoint.

This enables you to:
- Track character usage per custom tag across your organization
- Aggregate usage over a date range or break it down by day
- Paginate through large result sets

## Passing custom tags

To associate a request with a custom tag, include the `X-DeepL-Reporting-Tag` header in your API call. The header value accepts a single string that identifies the tag you want to assign to that request.

```bash
curl --request POST \
  --url https://api.deepl.com/v2/translate \
  --header 'Authorization: DeepL-Auth-Key YOUR_AUTH_KEY' \
  --header 'X-DeepL-Reporting-Tag: your-custom-tag' \
  --header 'Content-Type: application/json' \
  --data '{
    "text": ["Hello, world!"],
    "target_lang": "DE"
  }'
```

<Warning>
Custom tag data is currently supported for text translation and text improvement (rephrasing) only. Requests for document translation and speech-to-text are not yet tracked by custom tag. Support for additional request types will be added in a future update.
</Warning>
