# Docs Pipeline

Tools for detecting and fixing documentation gaps automatically.

## Gap Detection

Scans existing docs against the [IA standards](../standards/ia.yaml) and OpenAPI spec to find what's missing.

```bash
# Full audit
python pipeline/detect_gaps.py

# Audit one product family
python pipeline/detect_gaps.py --section translate

# Machine-readable output (for feeding into other tools)
python pipeline/detect_gaps.py --output json
```

Requires Python 3 and PyYAML (`pip install pyyaml`).

### What it checks

- **Product family coverage:** Does each product (Translate, Voice, Write, etc.) have an orientation page and at least one tutorial? Product families are auto-derived from OpenAPI spec tags.
- **Content quality:** Flags thin pages (< 100 words) and guides without code examples.
- **Frontmatter completeness:** Every docs-tab page should have a `description` field.
- **Ungrouped tags:** New OpenAPI tags that haven't been assigned to a product family yet.

### What it doesn't check

- Code samples (managed in a separate repo)
- API reference pages (auto-generated by Mintlify from the OpenAPI spec)
- Style or Diataxis compliance (that's the review agents' job)

## IA Standards

[`standards/ia.yaml`](../standards/ia.yaml) defines what "fully documented" looks like for any product section. It's intentionally high-level: what each section needs, not how to write each page.

The standards file does NOT define:
- Writing style (see `CLAUDE.md`)
- Diataxis content type rules (see `.claude/agents/diataxis.md`)
- A static list of every page path (the gap detection script discovers what exists)

### Adding a new product family

When a new product launches with new OpenAPI tags, the gap detection script will flag them as ungrouped. To assign them:

1. Decide: new top-level section, or grouped under an existing family?
2. Add an entry to `product_family_overrides` in `standards/ia.yaml`
3. Run `python pipeline/detect_gaps.py` to see what docs are needed
